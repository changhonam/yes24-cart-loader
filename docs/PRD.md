# PRD: Yes24 장바구니 일괄 담기 Chrome Extension

## 배경

독서동호회에서 도서 구매 신청 서비스(Streamlit 웹앱)를 운영 중이다.
회원들이 Yes24 도서 URL로 구매 신청하면, 관리자가 해당 도서들을 Yes24에서 일괄 구매한다.
현재는 관리자가 도서 URL을 하나씩 열어 장바구니에 담고 있으며, 이 과정을 자동화한다.

## 사용자

- **주 사용자**: 독서동호회 관리자 (1명)
- **사용 빈도**: 월 1~2회
- **처리 건수**: 회당 10~50권

## 핵심 요구사항

1. Yes24 도서 URL 목록을 입력받아 장바구니에 자동으로 일괄 담기
2. URL 목록은 Extension popup의 textarea에 붙여넣기로 입력 (줄바꿈 구분)
3. **같은 상품 URL이 여러 번 입력되면 자동으로 수량을 합산하여 1회 방문으로 담기** (Yes24가 중복 담기 시 수량을 합산하지 않는 동작에 대응)
4. Yes24에 이미 로그인된 브라우저 세션 활용
5. 요청 간 적절한 딜레이를 두어 차단 방지
6. 처리 결과를 항목별(종 단위)로 표시 (성공/실패/품절 등), 수량도 함께 표시

## 입력 형식

Streamlit 앱에서 복사한 줄바꿈 구분 텍스트. 다음 형식 모두 지원:

```
https://www.yes24.com/Product/Goods/59616782
http://www.yes24.com/Product/Goods/12345678?scode=032
http://www.yes24.com/24/goods/99999999
59616782
```

### URL 파싱 규칙

| 입력 형식 | 추출 방법 |
|-----------|-----------|
| `yes24.com/Product/Goods/{id}` | 정규식 `/yes24\.com\/Product\/Goods\/(\d+)/i` |
| `yes24.com/24/goods/{id}` (구형) | 정규식 `/yes24\.com\/24\/goods\/(\d+)/i` |
| 숫자만 (상품번호) | 정규식 `/^\d+$/` |
| 빈 줄/공백 | 무시 |
| 그 외 | "유효하지 않은 URL"로 무효 처리 (집계에서 제외) |

### 중복 URL 집계

같은 상품으로 인식된 URL(동일 `goodsId`)이 N번 등장하면, 배치 실행 시 해당 상품 페이지를 1회만 방문하고 **수량 N권**으로 장바구니에 담는다.

- 집계 키: `goodsId` (신형/구형 URL과 숫자 입력이 혼재해도 같은 ID면 합산)
- 무효 라인은 카운트되지만 배치 대상에서 제외

## Popup UI 와이어프레임

```
┌──────────────────────────────────┐
│  Yes24 장바구니 일괄 담기         │
│                                  │
│  도서 URL 입력 (한 줄에 하나):    │
│  ┌──────────────────────────────┐│
│  │ https://www.yes24.com/...    ││
│  │ 59616782                     ││
│  │                              ││
│  └──────────────────────────────┘│
│                                  │
│  입력 12줄 · 유효 10종 (14권)     │
│                                  │
│  [    담기 시작    ]  [ 중지 ]   │
│                                  │
│  진행: 5 / 10종 · 누적 7권        │
│  ████████░░░░░░░░░  50%         │
│                                  │
│  ┌──────────────────────────────┐│
│  │ ✓ 59616782 × 2권 - 성공      ││
│  │ ✓ 12345678 × 1권 - 성공      ││
│  │ ✗ 99999999 × 1권 - 품절      ││
│  │ ⏳ 22222222 × 3권 - 처리 중.. ││
│  │ ○ 33333333 × 1권 - 대기 중    ││
│  └──────────────────────────────┘│
│                                  │
│  결과: 성공 4종(7권) / 실패 1종  │
│        / 남은 5종 · 전체 10종    │
└──────────────────────────────────┘
```

### UI 상태 흐름

1. **입력 대기**: textarea 활성, "담기 시작" 버튼 활성
2. **처리 중**: textarea 비활성, "중지" 버튼 활성, 진행률 표시
3. **완료**: textarea 활성 (재입력 가능), 결과 요약 표시
4. **팝업 재열기**: 처리 중이면 현재 진행 상태 복원

## 에러 시나리오

| 상황 | 사용자 메시지 | 동작 |
|------|--------------|------|
| 유효하지 않은 URL | 무효 라인으로 카운트 표시 | 집계/배치에서 제외 |
| 상품 없음 (404) | "상품을 찾을 수 없습니다" | 실패 기록 |
| 품절 | "품절된 상품입니다" | 실패 기록 |
| Yes24 미로그인 | "Yes24에 먼저 로그인해주세요" | 배치 즉시 중단 |
| 페이지 로딩 타임아웃 | "페이지 로딩 시간 초과" | 실패 기록, 다음 항목 진행 |
| 장바구니 버튼 없음 | "장바구니 버튼을 찾을 수 없습니다" | 실패 기록 |
| 결과 감지 타임아웃 | "결과 확인 시간 초과" | 실패 기록 |
| 수량(qty>1) 수량증가 버튼 없음 | "수량 설정 실패: 수량증가 버튼을 찾을 수 없음" | 실패 기록 (qty=1이면 해당 단계 스킵) |
| 수량 반영 실패/제한 초과 | "수량 제한 초과 (요청 N, 반영 M)" | 실패 기록 |
| 네트워크 오류 | "네트워크 오류" | 실패 기록 |
